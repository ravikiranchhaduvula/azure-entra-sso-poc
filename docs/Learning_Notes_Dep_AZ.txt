set Django’s container port

(--Django runs inside the Docker container.
 --0.0.0.0:8000 means Django listens on port 8000 and accepts connections from outside the container.
 --127.0.0.1:8000 would accept connections only from inside the same container.
 --Azure App Service receives requests from the internet.
 --Azure must forward those requests to the port where Django is listening.
 --WEBSITES_PORT=8000 tells Azure to send traffic to container port 8000.
 --Therefore, Django’s port and Azure’s configured port must match.
 --Flow: Internet → Azure Web App → port 8000 → Django.
 --0.0.0.0 controls network accessibility; 8000 is the application port.
 --Without the correct host and port, the container may run but the website may not open.)

az webapp config appsettings set --resource-group capeark-auth-rg --name capeark-auth-ravi-2026 --settings WEBSITES_PORT=8000
